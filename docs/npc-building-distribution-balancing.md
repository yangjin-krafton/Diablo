# NPC 건물 분포 · 등급 밸런싱 설계

`home`은 플레이어 출생지 근처에 고정 배치한다. 그 외 NPC 건물은 시작
지점 주변에 몰아두지 않고, 건물 등급과 역할에 따라 행성 표면에 분산
배치한다.

목표는 초반 안전지대와 탐험 보상을 분리하는 것이다. 플레이어는 시작 직후
거점에서 기본 루프를 이해하고, 더 강한 성장 시설은 행성 위를 이동하며
찾아야 한다.

---

## 1. 핵심 규칙

| 대상 | 배치 규칙 | 이유 |
| ---- | --------- | ---- |
| `home` | 출생지 근처 고정 | 복귀, 퀘스트, 출발 준비의 기준점 |
| 일반 NPC 건물 | 중거리 분포 | 초반 탐험 동기 제공 |
| 희귀 NPC 건물 | 원거리 / 특정 바이옴 분포 | 빌드 성장의 목표 지점 |
| 전설 NPC 건물 | 행성당 0~1개 | 장기 목표, 행성 선택 가치 강화 |

`home`만 튜토리얼/안전권에 속한다. 검 강화 대장간, 마법 상점, 속성 검
시설 등은 모두 행성 분포 시스템의 대상이다.

---

## 2. 거리 밴드

거리는 `surface.arcDistance()` 기준의 행성 표면 거리로 계산한다.

| 밴드 | 출생지 기준 거리 | 용도 |
| ---- | ---------------- | ---- |
| Start | `0 ~ 7` | `home`, 시작 전투, 초반 드랍 |
| Near | `8 ~ 14` | 일반 건물, 쉬운 발견 |
| Mid | `15 ~ 24` | 주요 강화 건물, 적정 탐험 |
| Far | `25 ~ 38` | 희귀 건물, 위험 보상 |
| Edge | `39+` | 전설 건물, 보스/특수 이벤트 |

초기 권장값:

```js
npcDistribution: {
    startSafeRadius: 7,
    near: { min: 8, max: 14 },
    mid: { min: 15, max: 24 },
    far: { min: 25, max: 38 },
    edge: { min: 39, max: 52 },
}
```

## 3. NPC 건물 등급

| 등급 | 행성당 개수 | 거리 밴드 | 예시 | 목적 |
| ---- | ----------- | --------- | ---- | ---- |
| Common | 2~4 | Near / Mid | 상인, 회복소, 기본 수리 | 자주 만나는 보조 시설 |
| Uncommon | 1~3 | Mid | 일반 검 강화, 방어구 강화 | 핵심 성장 |
| Rare | 0~2 | Far | 속성 부여, 고급 스킬 트리 | 빌드 방향성 |
| Legendary | 0~1 | Edge / 조건부 | 번개 칼, 특수 유물, 전설 NPC | 행성별 특별 목표 |

현재 추가된 `swordUpgrade` 대장간은 **Uncommon** 기준이 적절하다.
시작지점 바로 옆이 아니라 Mid 밴드에 배치해, 플레이어가 어느 정도 전투와
드랍을 경험한 뒤 검술 스킬트리에 접근하게 한다.

---

## 4. 현재/예정 건물 매핑

| 건물 ID | 역할 | 등급 | 권장 밴드 | 비용/자원 축 |
| ------- | ---- | ---- | --------- | ------------ |
| `home` | 거점, 퀘스트, 출발 | Fixed | Start | 연료/출발 코어 |
| `swordUpgrade` | 일반 물리 검술 강화 | Uncommon | Mid | 5색 자원 중 1종 |
| `fireSwordUpgrade` | 화염 칼 강화 | Rare | Far | IGNIS 중심 |
| `lightningSwordUpgrade` | 번개 칼 강화 | Rare | Far | ARC 중심 |
| `elementForge` | 속성 변환/합성 | Legendary | Edge | 5색 복합 |
| `skillTrainer:*` | 장착 스킬별 전용 트리 | Uncommon~Rare | Mid / Far | 스킬 성격별 자원 |
| `statTrainer:*` | 캐릭터 기본 능력 강화 | Common~Rare | Near / Mid | 5색 자원 또는 공용 재료 |

물리 검은 첫 번째 무기 성장 축이므로 너무 멀면 안 된다. 반대로 화염 칼,
번개 칼 같은 속성 무기는 행성 탐험 보상으로 두어야 한다.

## 5. 장착 스킬 전용 트리 건물

하단 스킬바의 4개 장착 슬롯은 각각 별도의 스킬 성장 축을 가진다. 앞으로
장착 가능한 스킬이 계속 늘어나므로, 특정 스킬의 전용 트리를 열어주는
건물도 데이터 기반으로 확장해야 한다.

핵심 규칙:

- 장착 가능한 스킬마다 전용 `skillTrainer` 건물을 둘 수 있다.
- 행성마다 모든 스킬 트레이너가 반드시 등장하지 않는다.
- 현재 장착 중인 스킬의 트레이너는 등장 가중치를 올린다.
- 장착하지 않은 스킬의 트레이너도 낮은 확률로 등장해, 새 빌드로 갈아탈
  동기를 만든다.
- 하단 4개 슬롯 중 빈 슬롯이 있어도 건물 분포는 스킬 풀 전체 기준으로
  계산한다.

권장 등장 규칙:

| 조건 | 등장 가중치 |
| ---- | ----------- |
| 현재 장착 중인 스킬 | `×2.0` |
| 현재 행성 지배 원소와 스킬 원소가 일치 | `×1.6` |
| 이미 이번 행성에서 같은 스킬 트레이너 등장 | `×0.2` |
| 전설/상위 스킬 트리 | `×0.5` |

예시 매핑:

| 건물 ID | 대상 스킬 | 등급 | 권장 밴드 | 비용 축 |
| ------- | --------- | ---- | --------- | ------- |
| `skillTrainer:sword` | `sword` | Uncommon | Mid | 물리 + 5색 단일 |
| `skillTrainer:fireSword` | 화염 칼 | Rare | Far | IGNIS 중심 |
| `skillTrainer:lightningSword` | 번개 칼 | Rare | Far | ARC 중심 |
| `skillTrainer:shield` | 보호막 | Uncommon | Mid | GLACIA / BIOS |
| `skillTrainer:dash` | 이동기 | Common~Uncommon | Near / Mid | ARC 중심 |
| `skillTrainer:summon` | 소환 | Rare | Far | BIOS / TOXIN |

`swordUpgrade`는 현재 구현상 `skillTrainer:sword`의 첫 사례로 본다. 후속
작업에서는 건물 ID를 고정 클래스마다 늘리기보다, `skillId`를 가진 공통
스킬 트레이너 건물로 일반화하는 것이 좋다.

```js
skillTrainerBuildings: {
    sword: {
        targetSkillId: 'sword',
        modelPath: './asset/models/building/fig_s_blacksmith_shop.glb',
        rarity: 'uncommon',
        band: 'mid',
        element: 'physical',
        spawnChance: 0.75,
        maxPerPlanet: 1,
    },
}
```

행성당 스킬 트레이너 수 권장:

| 행성 단계 | 스킬 트레이너 총량 | 설명 |
| --------- | ------------------ | ---- |
| 초반 | 1~2 | 현재 장착 스킬 위주 |
| 중반 | 2~3 | 빌드 전환 후보 포함 |
| 후반 | 3~4 | 희귀/속성 스킬 트리 포함 |

스킬 트레이너가 없는 행성도 허용한다. 단, 2행성 연속으로 현재 장착 스킬의
트레이너가 하나도 나오지 않으면 다음 행성에서 보정 가중치를 주는 식으로
장기 운 실패를 줄인다.

## 6. 기본 스탯 강화 건물

캐릭터 기본 능력은 스킬과 별개로 성장한다. 각 기본 스탯마다 전용 NPC
건물 또는 시설을 두고, 해당 건물을 발견했을 때 자원을 지불해 영구 강화한다.

기본 스탯 건물은 너무 희귀하면 캐릭터 성장 체감이 약해진다. 초반 행성에도
1개 이상은 보이게 하고, 강력한 생존/경제 스탯은 중후반으로 밀어낸다.

### 6-1. 기본 스탯 건물 리스트

| 건물 ID | 강화 스탯 | 등급 | 권장 밴드 | 효과 예시 | 비용 축 |
| ------- | --------- | ---- | --------- | --------- | ------- |
| `statTrainer:maxHp` | 최대 HP | Common | Near / Mid | 랭크당 최대 HP +10~15 | BIOS / GLACIA |
| `statTrainer:hpRegen` | HP 자연 회복 | Uncommon | Mid | 랭크당 초당 회복 +0.2 | BIOS |
| `statTrainer:moveSpeed` | 이동 속도 | Common | Near / Mid | 랭크당 이동 속도 +3~5% | ARC |
| `statTrainer:pickupRange` | 재화 흡수 반경 | Common | Near | 랭크당 픽업 반경 +8~12% | ARC / BIOS |
| `statTrainer:armor` | 받는 피해 감소 | Uncommon | Mid | 랭크당 피해 감소 +3% | GLACIA |
| `statTrainer:damage` | 기본 공격력 | Uncommon | Mid | 모든 피해 +4~6% | IGNIS |
| `statTrainer:attackSpeed` | 공격/스킬 쿨타임 | Rare | Far | 쿨타임 -3~5% | ARC / IGNIS |
| `statTrainer:critChance` | 치명 확률 | Rare | Far | 치명 확률 +2~3% | TOXIN / IGNIS |
| `statTrainer:oreBonus` | 재화 획득량 | Rare | Far | 드랍 자원 +1 확률 | BIOS / TOXIN |
| `statTrainer:visionRange` | 감지/미니맵 반경 | Common | Near / Mid | 건물/위험 표시 반경 증가 | ARC / GLACIA |
| `statTrainer:revive` | 사망 보정 | Legendary | Edge | 세션당 1회 부활 또는 HP 1 생존 | 5색 복합 |

### 6-2. 스탯 성장 한도

기본 스탯은 무한히 올리면 밸런스가 무너지기 쉽다. 각 스탯마다 최대 랭크와
증가폭을 명확히 둔다.

| 스탯 | 권장 최대 랭크 | 누적 상한 |
| ---- | -------------- | --------- |
| 최대 HP | 10 | 기본 대비 +100~150 |
| HP 자연 회복 | 5 | 초당 +1.0 |
| 이동 속도 | 6 | 기본 대비 +25% 내외 |
| 재화 흡수 반경 | 5 | 기본 대비 +60% 내외 |
| 방어력 | 6 | 피해 감소 18~20% |
| 기본 공격력 | 8 | 모든 피해 +40% 내외 |
| 공격 속도/쿨타임 | 5 | 쿨타임 -20% 내외 |
| 치명 확률 | 5 | +15% 내외 |
| 재화 획득량 | 4 | 경제 과속 방지 |
| 감지 반경 | 4 | 탐험 편의성 중심 |

### 6-3. 등장 빈도

| 등급 | 행성당 개수 | 비고 |
| ---- | ----------- | ---- |
| Common 스탯 건물 | 1~3 | 초반 성장 체감 담당 |
| Uncommon 스탯 건물 | 1~2 | 생존/화력 방향 선택 |
| Rare 스탯 건물 | 0~1 | 고급 빌드 강화 |
| Legendary 스탯 건물 | 0~1 | 일부 행성에서만 등장 |

스탯 건물은 동일 행성에 같은 종류가 중복 등장하지 않는 편이 좋다. 대신
다른 행성에서 다시 만나 랭크를 올리는 구조가 탐험 루프와 잘 맞는다.

```js
statTrainerBuildings: {
    maxHp: {
        statId: 'maxHp',
        rarity: 'common',
        band: ['near', 'mid'],
        maxRank: 10,
        amountPerRank: 12,
        costElements: ['green', 'blue'],
        maxPerPlanet: 1,
    },
    pickupRange: {
        statId: 'pickupRange',
        rarity: 'common',
        band: ['near'],
        maxRank: 5,
        amountPerRank: 0.1,
        costElements: ['yellow', 'green'],
        maxPerPlanet: 1,
    },
}
```

## 7. 적대 건물 3종

일반 NPC/강화 건물과 별도로, 행성에는 적대 건물이 분포한다. 적대 건물은
플레이어에게 전투 목표, 위험 지역, 퀘스트 타깃, 대량 보상 지점을 제공한다.

| 건물 타입 | 행성당 개수 | 권장 밴드 | 역할 |
| --------- | ----------- | --------- | ---- |
| 요새형 | 1~3 | Mid / Far | 지역 점령, 경비 순찰, 파괴 목표 |
| 차원문형 | 3~6 | Near / Mid / Far | 보편 스폰 압력, 반복 봉쇄 목표 |
| 운송선형 | 이벤트 | 모든 밴드 | 예측 불가 난입, 순간 전투 밀도 상승 |

### 7-1. 요새형

요새형 건물은 고정된 적 거점이다. 요새 주변에는 경비형 몬스터가 주둔하고,
각 경비는 요새 기준 일정 반경 안에서 순찰한다. 플레이어가 감지 범위에
들어오면 순찰을 중단하고 추적/공격 상태로 전환한다.

| 항목 | 권장값 | 설명 |
| ---- | ------ | ---- |
| 주둔 경비 수 | 4~8 | 요새 등급에 따라 증가 |
| 순찰 반경 | `4 ~ 8` | 요새 중심 기준 |
| 감지 반경 | `7 ~ 12` | 유저 발견 시 공격 |
| 보충 쿨타임 | `12 ~ 25초` | 경비 사망 후 재생산 대기 |
| 최대 경비 수 | 설정값 고정 | 최대치 도달 시 생산 중지 |
| 요새 HP | 일반 건물보다 높음 | 유저 공격으로 파괴 가능 |
| 요새 자가 회복 | 매우 느림 | 전투 이탈 시 장기 복구 |

요새는 경비를 무한히 늘리지 않는다. `currentGuards < maxGuards`일 때만
보충 쿨타임이 돌고, 보충 가능한 슬롯이 없으면 생산을 멈춘다.

```js
fortress: {
    maxGuards: 6,
    patrolRadius: 6,
    aggroRadius: 10,
    reinforceCooldown: 18,
    hp: 450,
    regenPerSecond: 1.5,
    rewardTier: 'large',
}
```

요새 파괴는 퀘스트 목표로 사용할 수 있다.

- `요새 1개 파괴`
- `특정 원소 행성의 요새 파괴`
- `경비를 20마리 처치한 뒤 요새 파괴`
- `요새 파괴 후 드랍된 코어 회수`

보상은 일반 몬스터 처치보다 커야 한다. 권장 보상은 5색 자원 묶음, 대량
경험치, 희귀 강화 재료, 행성 출발 코어 보너스다.

### 7-2. 차원문형

차원문형 건물은 가장 보편적인 적 스폰 장치다. 행성마다 보통 3~6개가
존재하며, 일정 시간 간격으로 몬스터를 생성한다. 생성된 몬스터는 플레이어
위치를 알고 있는 상태로 출발해, 멀리서도 플레이어를 향해 추적한다.

| 항목 | 권장값 | 설명 |
| ---- | ------ | ---- |
| 행성당 개수 | 3~6 | 기본 전투 압력 |
| 소환 간격 | `10 ~ 22초` | 행성 난이도에 따라 감소 |
| 1회 소환 수 | 1~3 | 후반 행성에서 증가 |
| 활성 상태 | Open / Closed | HP 0이면 닫힘 |
| 재개방 시간 | `60 ~ 180초` | 닫힌 뒤 다시 열림 |
| 몬스터 AI | 전역 추적 | 생성 즉시 유저 위치를 목표로 설정 |

차원문은 파괴되어 사라지는 건물이 아니라, HP가 0이 되면 닫히는 구조다.
닫힌 차원문은 일정 시간이 지나면 다시 열리고, HP도 재개방 기준치까지
복구된다. 따라서 플레이어는 차원문을 영구 제거하는 것이 아니라, 행성
전투 압력을 일정 시간 낮추는 선택을 한다.

```js
portal: {
    spawnInterval: 16,
    spawnCount: 2,
    hp: 220,
    reopenDelay: 120,
    reopenedHpRatio: 0.6,
    spawnedEnemiesKnowPlayer: true,
}
```

차원문 관련 퀘스트 예시:

- `차원문 3개 닫기`
- `열린 차원문 근처 몬스터 15마리 처치`
- `차원문이 다시 열리기 전에 출발 준비 완료`
- `특정 원소 차원문 닫기`

차원문 보상은 요새보다 작지만 반복 가능해야 한다. 닫을 때마다 중간 보상을
주고, 퀘스트 완료 시 추가 보상을 지급한다.

### 7-3. 운송선형

운송선형은 고정 건물이 아니라 이벤트 스폰 방식이다. 게임 진행 중 하늘에서
유성처럼 행성 표면으로 떨어지고, 착탄 위치에서 한 번에 여러 몬스터가
그룹으로 등장한다. 플레이어가 미리 위치를 예측하기 어렵기 때문에, 순간적인
위험과 전투 밀도를 만드는 역할을 한다.

| 항목 | 권장값 | 설명 |
| ---- | ------ | ---- |
| 이벤트 간격 | `90 ~ 180초` | 세션 중 가끔 발생 |
| 사전 경고 | `2 ~ 4초` | 그림자, 궤적, 착탄 원 표시 |
| 착탄 피해 | 선택 | 너무 불합리하지 않게 낮게 |
| 1회 그룹 수 | 4~10 | 행성 난이도에 따라 증가 |
| 착탄 위치 | 플레이어 주변 중거리 | 너무 가까운 즉사 방지 |
| 재사용 | 즉시 소멸 | 고정 구조물로 남지 않음 |

운송선형은 "예측 불가"가 핵심이지만, 완전 무경고로 떨어지면 불공정하다.
짧은 경고 연출은 필요하다.

```js
dropShip: {
    eventCooldown: { min: 90, max: 180 },
    warningTime: 3,
    spawnCount: { min: 5, max: 8 },
    impactArcDistance: { min: 8, max: 16 },
    eliteChance: 0.12,
}
```

운송선 이벤트 보상은 즉시 지급보다 전투 결과 보상이 적합하다. 드랍 몬스터
중 마지막 개체를 처치했을 때 이벤트 클리어 보너스를 줄 수 있다.

## 8. 적대 건물 밸런싱 기준

| 타입 | 위험도 | 지속성 | 보상 | 퀘스트 적합도 |
| ---- | ------ | ------ | ---- | ------------- |
| 요새형 | 높음 | 파괴 전까지 지속 | 큼 | 매우 높음 |
| 차원문형 | 중간 | 닫혀도 재개방 | 중간 / 반복 | 높음 |
| 운송선형 | 순간 높음 | 일회성 | 중간 | 이벤트형 |

세 타입은 서로 다른 압력을 만든다.

- 요새형: 지역을 점령하고 탐험 경로를 막는다.
- 차원문형: 행성 전체에 지속적인 스폰 압력을 만든다.
- 운송선형: 안전하다고 느끼는 순간 전투 변수를 만든다.

한 행성에서 세 타입이 동시에 과하면 피로도가 높아진다. 초반 행성은
차원문형 중심, 중반부터 요새형, 후반부터 운송선 이벤트 빈도를 높이는
방식이 좋다.

초기 권장 행성별 구성:

| 행성 단계 | 요새형 | 차원문형 | 운송선 이벤트 |
| --------- | ------ | -------- | ------------- |
| 초반 | 0~1 | 3 | 낮음 |
| 중반 | 1~2 | 4 | 보통 |
| 후반 | 2~3 | 5~6 | 높음 |

## 9. 행성 편향과 건물 분포

행성의 지배 원소(`CONFIG.planets[*].dominant`)는 드랍률뿐 아니라 NPC 건물
등장 확률에도 영향을 준다.

| 지배 원소 | 등장 확률 증가 건물 |
| --------- | ------------------ |
| `red` / IGNIS | 화염 칼, 공격력, 폭발 강화, 화염 포탑 |
| `yellow` / ARC | 번개 칼, 이동 속도, 공격 속도, 체인 공격 |
| `green` / BIOS | 최대 HP, 회복, 흡혈, 생존 패시브 |
| `blue` / GLACIA | 방어력, 둔화, 보호막, 방어 시설 |
| `purple` / TOXIN | 치명, 재화 보너스, 독, 부식, 지속 피해 |

예시:

```js
spawnWeight(building, planet) =
    baseWeight(building.rarity)
    * rarityDistanceWeight(building.rarity, distanceBand)
    * elementBiasWeight(building.element, planet.dominant)
```

속성 건물은 자기 원소 행성에서 더 자주 나오되, 다른 행성에서도 낮은
확률로 등장할 수 있다. 이렇게 해야 행성 선택이 빌드 선택으로 이어진다.

## 10. 배치 알고리즘 권장안

1. 출생지와 `home` 위치를 먼저 확정한다.
2. `startSafeRadius` 안에는 `home` 외 NPC 건물을 배치하지 않는다.
3. 등급별 후보 건물 풀을 만든다.
4. 각 건물의 등급에 맞는 거리 밴드에서 임의 지점을 뽑는다.
5. 기존 건물/랜드마크/보스 지점과 최소 간격을 검사한다.
6. 행성 지배 원소와 건물 원소가 맞으면 가중치를 올린다.
7. 실패 시 같은 밴드에서 재시도하고, 일정 횟수 실패하면 한 단계 먼 밴드로
   밀어낸다.

최소 간격 권장값:

| 대상 | 최소 간격 |
| ---- | --------- |
| NPC 건물 ↔ NPC 건물 | `6` |
| NPC 건물 ↔ home | `8` |
| Rare 이상 ↔ Rare 이상 | `12` |
| Legendary ↔ 보스 지점 | `10` |
| 요새형 ↔ home | `14` |
| 차원문형 ↔ home | `10` |
| 요새형 ↔ 요새형 | `18` |

## 11. 플레이 리듬 기준

| 시간 | 기대 경험 |
| ---- | --------- |
| 0~1분 | `home` 발견, 이동/전투 이해 |
| 1~3분 | 첫 기본 스탯 건물 또는 대장간 힌트 발견 |
| 3~6분 | `swordUpgrade` 또는 장착 스킬 트레이너 접근 |
| 6~10분 | Rare 건물 탐색 또는 출발 준비 |
| 10분+ | Edge 건물/보스/다음 행성 목표 |

`swordUpgrade`는 첫 세션에서 너무 늦게 나오면 성장 루프가 보이지 않는다.
Mid 밴드에 1개는 거의 확정으로 배치하는 것이 좋다.

## 12. 코드 작업 메모

- `CONFIG.home.spawnArcOffset`은 유지한다. `home`은 예외적으로 시작지점 근처.
- `CONFIG.swordUpgrade.spawnArcOffset` 같은 단일 오프셋은 임시값으로만 두고,
  후속 작업에서 등급 기반 분포 테이블로 이동한다.
- 추천 신규 설정:
  ```js
  npcBuildings: {
      swordUpgrade: {
          modelPath: './asset/models/building/fig_s_blacksmith_shop.glb',
          rarity: 'uncommon',
          role: 'weapon',
          element: 'physical',
          count: { min: 1, max: 1 },
          band: 'mid',
          interactRange: 2.8,
      },
      statTrainerMaxHp: {
          rarity: 'common',
          role: 'stat',
          statId: 'maxHp',
          band: 'near',
          interactRange: 2.8,
      },
  }
  ```
- 추천 신규 시스템:
  - `src/js/systems/npc-building-placement.js`
  - 등급/밴드/행성 편향을 받아 NPC 건물의 위치를 생성
  - `NpcBase.place()`는 고정 배치가 아니라 배치 시스템에서 받은 위치를
    적용할 수 있게 확장
  - 현재 장착 스킬 4개를 읽어 `skillTrainer:*` 등장 가중치에 반영
  - 기본 스탯 랭크는 별도 `PlayerStatsProgression` 또는 저장 가능한
    컨트롤러에서 관리
- 추천 적대 건물 시스템:
  - `src/js/systems/hostile-building-system.js`
  - 요새 경비 보충, 차원문 재개방, 운송선 이벤트 쿨타임 관리
  - 적 스폰은 기존 `Spawner`를 재사용하되, 스폰 출처와 AI 목표를 명시
  - 건물 HP/회복/파괴 보상은 공통 `HostileBuilding` 베이스로 분리
- `home-npc.js`만 고정 `place()`를 유지하고, 나머지 NPC는 공통 분포 로직을
  사용한다.

## 13. 밸런싱 원칙

- 시작지점 근처에는 `home`만 둔다.
- 첫 무기 성장 시설은 세션 초반에 발견 가능해야 한다.
- 장착 스킬 트레이너는 랜덤 등장하되, 현재 장착 스킬을 너무 오래 못
  만나지 않게 보정한다.
- 기본 스탯 건물은 초반 성장 체감을 위해 Common 등급을 충분히 배치한다.
- 속성 무기 시설은 행성 테마와 연결한다.
- 희귀 건물은 멀수록 강한 보상을 제공한다.
- 한 화면 안에 여러 NPC 건물이 몰리지 않게 최소 간격을 둔다.
- 행성마다 "이번 행성에서 기대할 수 있는 성장 방향"이 달라야 한다.
- 적대 건물은 보상 없이 피로도만 주면 안 된다.
- 차원문형은 반복 압력, 요새형은 목표 지점, 운송선형은 이벤트 변수로
  역할을 분리한다.

# 신규 스킬 제안

하단 스킬바는 4개의 장착 슬롯을 가진다. 스킬은 앞으로 계속 확장될 수
있으므로, 각 스킬은 다음 기준을 만족해야 한다.

- 자동 전투 루프와 잘 맞아야 한다.
- 이동만 직접 조작하는 플레이에서도 의미가 있어야 한다.
- 전용 스킬트리와 전용 `skillTrainer:*` 건물을 가질 수 있어야 한다.
- 5색 자원(`red/yellow/green/blue/purple`) 중 1~2개와 성장 방향이 연결되어야
  한다.
- 한 스킬이 모든 상황을 해결하지 않고, 다른 스킬과 조합할 여지를 남겨야
  한다.

---

## 1. 스킬 분류

| 분류 | 역할 | 예시 |
| ---- | ---- | ---- |
| 무기형 | 기본 공격 루프를 만든다 | 검술, 화염 칼, 번개 칼 |
| 방어형 | 생존 시간을 늘린다 | 보호막, 반격, 회복장 |
| 이동형 | 위치 선정과 회피를 돕는다 | 돌진, 순간이동, 가속장 |
| 광역형 | 다수 몬스터를 정리한다 | 폭발, 낙뢰, 독구름 |
| 소환형 | 전투 보조 유닛을 만든다 | 드론, 정령, 포탑 |
| 제어형 | 적의 행동을 늦추거나 묶는다 | 빙결, 끌어당김, 공포 |

초기 확장은 무기형 1개, 방어형 1개, 이동형 1개, 광역형 1개 정도가 좋다.
플레이어가 4칸을 서로 다른 역할로 채울 수 있어야 빌드 선택이 생긴다.

## 3. 스킬 상세안

### 3-1. `dash` 전투 돌진

짧은 거리로 빠르게 돌진한다. 자동 스킬로 둘 경우, 적에게 둘러싸였거나
위험도가 높을 때 가장 가까운 안전 방향으로 발동한다. 수동 입력이 추가되면
플레이어 진행 방향으로 발동한다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 이동형 |
| 기본 쿨타임 | `5 ~ 7초` |
| 주 자원 | ARC |
| 보조 자원 | IGNIS |
| 전용 건물 | `skillTrainer:dash` |

스킬트리 방향:

- 돌진 거리 증가
- 쿨타임 감소
- 돌진 중 무적 시간
- 돌진 경로의 적에게 피해
- 돌진 후 다음 검 공격 강화

### 3-2. `barrier` 에너지 보호막

일정 시간 동안 피해를 흡수하는 보호막을 생성한다. 자동 발동 조건은 HP가
일정 비율 이하이거나, 주변 적 수가 많을 때로 잡는다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 방어형 |
| 기본 쿨타임 | `12 ~ 18초` |
| 주 자원 | GLACIA |
| 보조 자원 | BIOS |
| 전용 건물 | `skillTrainer:barrier` |

스킬트리 방향:

- 보호막 흡수량 증가
- 지속 시간 증가
- 보호막 파괴 시 폭발
- 보호막 중 이동 속도 증가
- 보호막 일부를 HP 회복으로 전환

### 3-3. `fireNova` 화염 폭발

플레이어 주변에 원형 폭발을 일으켜 근접한 적을 정리한다. 검술이 전방 공격
중심이라면, 화염 폭발은 포위 상황을 해결하는 역할을 한다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 광역형 |
| 기본 쿨타임 | `8 ~ 12초` |
| 주 자원 | IGNIS |
| 보조 자원 | ARC |
| 전용 건물 | `skillTrainer:fireNova` |

스킬트리 방향:

- 폭발 반경 증가
- 피해 증가
- 화상 지속 피해
- 적 넉백
- 폭발 후 짧은 화염 지대 생성

### 3-4. `chainLightning` 연쇄 번개

가장 가까운 적에게 번개를 떨어뜨리고, 주변 적에게 연쇄된다. 적이 흩어져
있을 때 유리하며, 공격 속도/쿨타임 빌드와 잘 맞는다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 광역형 |
| 기본 쿨타임 | `6 ~ 10초` |
| 주 자원 | ARC |
| 보조 자원 | GLACIA |
| 전용 건물 | `skillTrainer:chainLightning` |

스킬트리 방향:

- 연쇄 횟수 증가
- 연쇄 거리 증가
- 같은 대상 재타격 허용
- 짧은 기절
- 치명타 시 추가 연쇄

### 3-5. `toxicCloud` 부식 구름

지속 피해 구역을 남긴다. 차원문, 요새 경비, 느린 보스처럼 오래 머무르는
적에게 강하다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 제어 / 지속 피해 |
| 기본 쿨타임 | `10 ~ 14초` |
| 주 자원 | TOXIN |
| 보조 자원 | BIOS |
| 전용 건물 | `skillTrainer:toxicCloud` |

스킬트리 방향:

- 구름 지속 시간 증가
- 반경 증가
- 방어력 감소 디버프
- 중독 중인 적 처치 시 확산
- 보스에게 낮은 효율의 중첩 피해

### 3-6. `healingPulse` 생체 파동

주기적으로 HP를 회복하거나, 위험할 때 즉시 회복한다. 회복만 제공하면
수동성이 강하므로, 과치유 일부를 보호막으로 바꾸는 트리가 필요하다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 방어 / 회복 |
| 기본 쿨타임 | `16 ~ 24초` |
| 주 자원 | BIOS |
| 보조 자원 | GLACIA |
| 전용 건물 | `skillTrainer:healingPulse` |

스킬트리 방향:

- 회복량 증가
- 쿨타임 감소
- 과치유 보호막
- 주변 소환수/드론 회복
- 회복 후 짧은 피해 감소

### 3-7. `iceField` 빙결 지대

플레이어 주변 또는 적 밀집 지역에 둔화 지대를 만든다. 생존과 광역 스킬
연계를 돕는다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 제어형 |
| 기본 쿨타임 | `12 ~ 18초` |
| 주 자원 | GLACIA |
| 보조 자원 | ARC |
| 전용 건물 | `skillTrainer:iceField` |

스킬트리 방향:

- 둔화율 증가
- 지속 시간 증가
- 일정 시간 머문 적 빙결
- 빙결 대상 추가 피해
- 지대 안에서 플레이어 방어력 증가

### 3-8. `turretDrone` 전투 드론

플레이어 주변을 따라다니며 자동 사격하는 보조 유닛을 생성한다. 구현 난도는
다소 높지만, 소환형 빌드의 기반이 된다.

| 항목 | 값 |
| ---- | -- |
| 분류 | 소환형 |
| 기본 쿨타임 | `20 ~ 30초` |
| 주 자원 | ARC |
| 보조 자원 | BIOS |
| 전용 건물 | `skillTrainer:turretDrone` |

스킬트리 방향:

- 드론 지속 시간 증가
- 동시 드론 수 증가
- 사거리 증가
- 관통 사격
- 드론 사망 시 폭발

## 4. 속성 검 확장

현재 검술은 일반 물리 검을 기준으로 한다. 이후 화염 칼, 번개 칼은 별도
스킬 또는 검술의 상위 변형으로 확장할 수 있다.

| 스킬 ID | 이름 | 처리 방식 | 주 자원 | 특징 |
| ------- | ---- | --------- | ------- | ---- |
| `sword` | 일반 검술 | 기본 무기형 | 물리 / 5색 단일 | 안정적 근접 공격 |
| `fireSword` | 화염 칼 | 별도 스킬 또는 변형 트리 | IGNIS | 폭발, 화상, 광역 |
| `lightningSword` | 번개 칼 | 별도 스킬 또는 변형 트리 | ARC | 공격 속도, 연쇄, 추적 |

권장 방향은 초반에는 `sword` 하나만 두고, 이후 속성 검은 별도
`skillTrainer:fireSword`, `skillTrainer:lightningSword` 건물을 통해 해금하는
방식이다. 이렇게 하면 행성 탐험과 무기 빌드 확장이 자연스럽게 연결된다.

## 5. 스킬 장착 규칙

초기 규칙:

- 하단 슬롯은 4개.
- 같은 스킬 중복 장착은 금지.
- 무기형 스킬은 1~2개까지만 권장.
- 방어형 또는 이동형 없이 광역형만 4개 장착하면 생존이 불안정해야 한다.
- 스킬 자동 발동 조건은 스킬별로 다르게 둔다.

권장 자동 발동 조건:

| 분류 | 발동 조건 |
| ---- | --------- |
| 무기형 | 적이 사거리 안에 있음 |
| 방어형 | HP 낮음, 피격 직전, 주변 적 많음 |
| 이동형 | 포위됨, 위험 반경 진입, 수동 입력 |
| 광역형 | 주변 적 수가 일정 이상 |
| 소환형 | 소환수 없음 또는 지속 시간 종료 |
| 제어형 | 적 밀집도 높음, 요새/차원문 근처 |

## 6. 스킬 트레이너 건물 연결

각 스킬은 전용 트레이너 건물과 연결된다.

```js
skillDefinition: {
    id: 'fireNova',
    displayName: '화염 폭발',
    category: 'aoe',
    element: 'red',
    trainerBuildingId: 'skillTrainer:fireNova',
}
```

트레이너 건물은 행성마다 랜덤 등장한다. 현재 장착 중인 스킬의 건물은
가중치를 올리고, 행성 지배 원소와 맞는 스킬은 추가 가중치를 준다.

| 상황 | 처리 |
| ---- | ---- |
| 현재 장착 스킬 | 등장 확률 증가 |
| 해당 행성 지배 원소와 일치 | 등장 확률 증가 |
| 이미 최대 랭크 달성 | 등장 확률 감소 |
| 최근 2행성 동안 못 만남 | 보정 등장 확률 증가 |

## 7. 구현 우선순위

| Phase | 스킬 | 이유 |
| ----- | ---- | ---- |
| 1 | `dash` | 이동 조작 재미와 생존 선택지 증가 |
| 2 | `barrier` | 초반 생존 안정화 |
| 3 | `fireNova` | 포위 상황 대응, 화염 행성 연계 |
| 4 | `chainLightning` | ARC 행성/번개 칼 확장 기반 |
| 5 | `toxicCloud` / `iceField` | 제어/DoT 빌드 확장 |
| 6 | `turretDrone` | 소환형 시스템 기반 |

## 8. 코드 작업 메모

- `src/js/skills/skill-system.js`
  - 장착 가능한 스킬 풀과 4개 슬롯을 분리한다.
  - 현재는 `SwordSkill + EmptySkill` 구조지만, 이후 `availableSkills`와
    `equippedSkillIds`를 분리하는 것이 좋다.
- `src/js/skills/*`
  - 각 스킬은 `Skill` 베이스를 상속하고 `getNodes()`로 전용 트리를 가진다.
  - 자동 발동 조건은 스킬 클래스 내부에서 관리한다.
- `src/js/npc/skill-tree-panel.js`
  - 이미 `skillId` 기반으로 특정 스킬 트리를 열 수 있다.
  - `skillTrainer:*` 건물은 이 패널을 재사용하면 된다.
- `src/js/npc/*`
  - 스킬별 건물 클래스를 계속 늘리기보다, `targetSkillId`를 가진 공통
    `SkillTrainerNpc`로 일반화하는 방향이 좋다.
- `docs/npc-building-distribution-balancing.md`
  - 스킬 트레이너 건물의 행성 분포 규칙과 연결된다.

## 9. 밸런싱 원칙

- 스킬은 강함보다 역할이 먼저 구분되어야 한다.
- 자동 발동만으로도 납득 가능한 행동을 해야 한다.
- 같은 원소 스킬만 장착하면 특정 행성에서는 강하지만, 다른 행성에서는
  자원 수급이 불안정해야 한다.
- 방어/이동 스킬은 피해량이 낮아도 빌드 가치가 있어야 한다.
- 광역 스킬은 강하지만 쿨타임과 위치 조건으로 제어한다.
- 소환형 스킬은 화면 혼잡과 성능 비용을 항상 고려한다.

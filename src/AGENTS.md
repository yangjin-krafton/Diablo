# AGENTS.md

이 문서는 `src/` 하위 전체 코드에 적용된다.

## Project Shape

- 이 디렉터리는 별도 번들러 없이 `src/index.html`에서 import map으로 `three`와 `pixi.js`를 불러오는 정적 웹 게임이다.
- `src/js/main.js`는 진입점이고, `src/js/game.js`가 렌더러, 씬, 카메라, 입력, 플레이어, NPC, 스포너, 드롭, 스킬, UI를 조립한다.
- 3D 월드와 게임 루프는 Three.js 기반이고, HUD/패널/스킬바는 Pixi.js 오버레이(`UIRoot`)로 렌더링한다.
- 튜닝 값은 가능한 한 `src/js/config.js`의 `CONFIG`를 단일 출처로 사용한다.

## Directory Guide

- `asset/`: 아이콘과 GLB 모델 등 정적 에셋.
- `css/`: HTML과 캔버스 주변의 기본 레이아웃 CSS. Pixi 기반 게임 UI는 보통 JS에서 그린다.
- `js/combat/`: 공격 판정, 검기, 스윙 같은 전투 표현과 런타임 로직.
- `js/entities/`: 플레이어, 적, 드롭 아이템처럼 월드에 배치되는 엔티티.
- `js/npc/`: NPC/건물과 이들이 여는 Pixi 패널. 현재 홈 패널과 스킬 트리 패널이 여기에 있다.
- `js/skills/`: 스킬 정의, 레벨/경험치/포인트, 런타임 쿨다운과 효과.
- `js/systems/`: 스포닝, 드롭, 홈 퀘스트/출발 같은 게임 규칙 시스템.
- `js/ui/`: 전역 UI 루트와 항상 표시되는 UI(예: 스킬바).
- `js/world/`: 구형 표면, 지형, 월드 배치 유틸리티.

## Coding Rules

- ES modules를 유지하고, import 경로는 현재 스타일처럼 상대 경로와 `.js` 확장자를 명시한다.
- 전역 상태를 새로 만들기보다 `Game`에서 시스템을 조립하고 필요한 의존성을 생성자 인자로 전달한다.
- 프레임마다 생성되는 객체 수를 줄인다. 반복 호출되는 update/render 경로에서는 가능하면 모듈 스코프의 임시 `THREE.Vector3`, `THREE.Quaternion` 등을 재사용한다.
- 플레이어는 화면 상단의 고정 위치에 두고, 이동은 `worldRotator`를 누적 회전시키는 현재 모델을 유지한다.
- 게임 밸런스 숫자, 모델 경로, 반경, 쿨다운, 확률 등은 하드코딩하지 말고 `CONFIG`에 추가한다.
- 사망 후에도 유지되어야 하는 성장 데이터와, 재시작 시 초기화되어야 하는 런타임 데이터(쿨다운, 액티브 메시, 드롭, 적)는 구분한다.
- 기존 파일의 간단한 클래스 기반 구조를 따른다. 큰 프레임워크나 빌드 도구를 추가하지 않는다.

## UI Rules

- Pixi UI는 `UIRoot`의 레이어를 사용한다: HUD는 `hudLayer`, 모달 패널은 `panelLayer`, 하단 바는 `barLayer`.
- Pixi 캔버스는 기본적으로 `pointer-events: none`이며, hit test로 상호작용 가능한 UI 위에서만 포인터 이벤트를 받는다. 새 UI도 이 흐름을 깨지 않는다.
- 패널은 resize 콜백(`uiRoot.onResize`)에서 화면 크기에 맞게 레이아웃을 다시 계산한다.
- 모바일 화면을 고려해 고정 폭만 쓰지 말고 `Math.min`, 여백 상수, wordWrap, 텍스트 축소 유틸리티를 사용한다.
- 전투 중 항상 표시되는 UI와 NPC/건물에서 여는 패널을 결합하지 않는다. NPC/건물 전용 패널은 `js/npc/` 쪽에 둔다.

## Gameplay Rules

- 홈 근처 진입 시 일시정지하고 홈 패널을 열며, 닫은 뒤에는 범위를 나갔다가 다시 들어와야 재오픈되는 현재 hysteresis 동작을 유지한다.
- 최종 출발 카운트다운, 퀘스트 진행도, 연료 적재 같은 홈 상태는 `HomeController`가 관리한다.
- 적 사망 보상은 `DropSystem`과 `SkillSystem`의 계약을 통해 처리한다. 스포너나 적 엔티티에 성장 로직을 직접 넣지 않는다.
- 스킬 슬롯 UI는 장착/발동 상태를 보여주는 역할이고, 스킬 트리의 포인트 배분/리셋은 스킬 또는 스킬 트리 패널 쪽 책임으로 둔다.

## Verification

- 정적 실행은 `src/index.html`을 브라우저에서 열어 확인한다. CDN import map을 사용하므로 네트워크가 필요하다.
- 변경 후 최소 확인 항목:
  - 로딩 화면이 사라지고 3D 씬이 렌더링되는지.
  - WASD/방향키와 모바일 드래그 입력이 이동을 깨지 않는지.
  - 적 스폰, 자동 공격, 드롭 획득, 스킬 경험치가 동작하는지.
  - 홈 NPC 진입/패널 열기/닫기/퀘스트/연료/출발 카운트다운 흐름이 동작하는지.
  - Pixi UI가 Three.js 캔버스 위에서 클릭 영역을 올바르게 처리하는지.

# 5색 드랍 자원 · 재화 시스템 설계

행성 표면에서 채집되는 5종 원소 결정을 게임의 핵심 재화로 사용한다.
*우주 생존 / 거점 탈출* 테마에 맞춰 드랍 → 픽업 → 적재 → 업그레이드의
순환 루프를 설계한다.

> 코드 매핑 — `home-controller.js`의 `ores` 키와 일치
> `red / yellow / green / blue / purple` × 5

---

## 1. 컨셉 개요

- 플레이어는 미지의 행성에 불시착한 생존자.
- 거점(`home`)은 본진이자 *재출항 가능한 셔틀/정거장*.
- 적은 행성 토착 생물 + 결정에 잠식된 침입체.
- 결정은 행성 곳곳의 원소 에너지가 응축된 부산물 → 적이 흡수해 지니고
  있다가 처치 시 드랍.
- 메인 미션 "거점 출발"은 단순 연료 적재가 아니라 *5종 결정의 균형 잡힌
  코어 충전*으로 확장된다.
- **세션 단위 = 행성 1개**. 행성을 옮겨 다니며 코어를 채우는 *행성 순회
  파밍 루프*가 메타 진척의 뼈대.

---

## 2. 색 → 원소 매핑

| 키       | 한글 라벨 | 코드명 | 원소 테마        | 기조 색           | 시각 모티프                              |
| -------- | --------- | ------ | ---------------- | ----------------- | ---------------------------------------- |
| `red`    | 적        | IGNIS  | 불 · 플라즈마    | `#ff3f5f` 주홍    | 일렁이는 코어, 적색 입자 트레일          |
| `yellow` | 황        | ARC    | 전기 · 광자      | `#ffd84f` 황금    | 정전기 스파크, 격자형 라인               |
| `green`  | 녹        | BIOS   | 자연 · 생체      | `#62f59a` 형광녹  | 포자/덩굴, 박동하는 유기체 결정          |
| `blue`   | 청        | GLACIA | 물 · 냉각        | `#33c9ff` 시안    | 결빙 균열, 차가운 안개                   |
| `purple` | 자        | TOXIN  | 독 · 부식 가스   | `#b469ff` 보라    | 흩어지는 유독 연무, 수액                 |

### 한 줄 정의

- **IGNIS** — *분출하는 화염 코어.* 폭발/관통/돌격에 쓰이는 공격적 자원.
- **ARC** — *축적된 전기 입자.* 사거리/속도/체이닝에 쓰이는 정밀 자원.
- **BIOS** — *살아있는 결정.* 회복/지속력/소환에 쓰이는 유지 자원.
- **GLACIA** — *압축된 냉각수.* 둔화/쉴드/광역 제어 자원.
- **TOXIN** — *부식 가스.* 디버프/지속피해/방어 무력화 자원.

## 3. 드랍 설계

### 3-1. 기본 규칙

| 항목         | 규칙                                                                  |
| ------------ | --------------------------------------------------------------------- |
| 드랍 트리거  | 적 사망 시 1회 롤                                                     |
| 드랍 확률    | 일반 30%, 엘리트 80%, 보스 100% + 보너스                              |
| 드랍 종류 수 | 1체당 1종 (보스는 2~3종 동시)                                         |
| 픽업 방식    | 일정 거리 진입 시 자동 흡수 (`drops.pickupRange`)                     |
| 시각 표현    | 결정형 메시 + 색상 글로우 + 살짝 떠오르는 idle 모션                  |

### 3-2. 적 타입별 드랍 풀

각 적 타입에 *주 드랍*과 *부 드랍*을 설정해 자연스러운 자원 분포를 만든다.

| 적 타입         | 주 드랍 (60%)  | 부 드랍 (30%) | 기타 (10%)    |
| --------------- | -------------- | ------------- | ------------- |
| 화염 토착체     | IGNIS          | ARC           | TOXIN         |
| 전기 갑각충     | ARC            | GLACIA        | IGNIS         |
| 식생 잠식체     | BIOS           | TOXIN         | GLACIA        |
| 한기 침입체     | GLACIA         | BIOS          | ARC           |
| 부식 포자체     | TOXIN          | BIOS          | IGNIS         |
| 보스            | 모든 색 균등 + 추가 보너스 (확정 1+α)                          |

### 3-3. 바이옴 가중치 (선택 확장)

`world/terrain.js`의 바이옴별로 드랍 풀에 ±20% 가중치를 곱해 *바이옴별
주력 자원* 개념을 만든다. (예: 용암 지대 → IGNIS↑ ARC↑ GLACIA↓)

### 3-4. 행성 드랍 편향 — 핵심 메타 루프

행성마다 *지배 원소(dominant element)* 가 있고, 해당 색의 드랍률이
압도적으로 높아진다. 한 자원만 빠르게 모으고 싶다면 *그 색 행성*을
방문해야 한다.

| 행성       | 지배 원소 | 환경 컨셉                          | 액센트 색 (지배) |
| ---------- | --------- | ---------------------------------- | ---------------- |
| 화염 행성  | IGNIS     | 용암 분출, 검은 현무암, 붉은 하늘  | `#ff3f5f` 주홍   |
| 폭풍 행성  | ARC       | 사막 + 영구 번개 폭풍, 정전기 안개 | `#ffd84f` 황금   |
| 정글 행성  | BIOS      | 거대 식생, 포자 안개, 형광 코어    | `#62f59a` 형광녹 |
| 빙하 행성  | GLACIA    | 영구 결빙, 깊은 균열, 푸른 안개    | `#33c9ff` 시안   |
| 부식 행성  | TOXIN     | 유독 늪, 자줏빛 가스, 썩은 결정    | `#b469ff` 보라   |

> 표의 *액센트 색*은 *지배 원소 단일 색*. 실제 환경 톤(하늘/지형/안개)은
> 다음 3-5 절의 *비율 합성* 으로 자동 도출되므로 행성마다 더 미묘한
> 차이가 생긴다.

#### 드랍 가중치 공식

행성에 입성하면 적의 기본 드랍 풀 위에 *행성 편향(Planet Bias)* 이
곱해진다.

```
finalChance(color) = baseDropChance(color) × planetBias[color]

planetBias on 화염 행성:
  IGNIS  ×3.0   ← 지배 원소
  ARC    ×0.8
  BIOS   ×0.6
  GLACIA ×0.5
  TOXIN  ×0.7
```

지배 색은 ×3.0 이상으로 끌어올리고, 나머지는 ×0.5~0.8로 깎아 *모든
자원이 다 떨어지진 않게* 한다 (변환 1~2회로 보충 가능한 양).

#### 결과적 드랍 비율 예시 (화염 행성)

| 자원   | 비율  |
| ------ | ----- |
| IGNIS  | ~58%  |
| ARC    | ~10%  |
| BIOS   | ~8%   |
| GLACIA | ~7%   |
| TOXIN  | ~9%   |
| 미드랍 | ~8%   |

→ 대략 *행성 색의 자원이 절반 이상*, 나머지는 골고루.

#### 메타 루프 — 행성 순회 파밍

1. 화염 행성 착륙 → IGNIS 집중 수급 + 일부 부재화.
2. 출발 코어 준비 → 다음 색 행성으로 워프.
3. 다음 행성에서 부족한 색 보충 → 원하는 코어 빌드 완성.
4. 5종 균형이 맞으면 *최종 목적지 / 보스 행성* 해금.

행성 선택 자체가 *플레이 빌드*가 된다 — 화염 검 빌드를 좋아하면 화염
행성에서 오래 머물고, 균형 잡힌 빌드는 5색 행성을 짧게 순회.

#### 행성 간 이동 비용

- 출발 카운트다운 통과 = 다음 행성 워프.
- 워프 시 *행성 선택 화면*에서 비용 결정:
  - 인접 색 행성: 코어 1개 소모
  - 반대편 색 행성: 코어 2~3개 소모 (먼 거리)
- 보유 자원에 따라 *갈 수 있는 행성*이 달라지므로 자원 관리가 곧
  여행 전략.

### 3-5. 행성 팔레트 합성 (드랍 비율 = 시각 정체성)

행성마다의 *고유 환경 컬러 팔레트*는 별도로 손으로 잡지 않고, **드랍
편향(`planetBias`)을 그대로 가중치로 사용해 5색을 비율 합성**하여
자동으로 산출한다.

> 같은 데이터(편향)가 *게임플레이 드랍률*과 *시각적 정체성* 양쪽을
> 동시에 결정 → 한 곳을 튜닝하면 양쪽이 함께 따라온다. 플레이어는
> 환경 톤을 *읽고* 그 행성에서 무엇이 주로 떨어지는지 직관적으로
> 추측할 수 있다.

#### 합성 공식

```
ELEMENT_HEX = {
    red:    0xff3f5f,    // IGNIS
    yellow: 0xffd84f,    // ARC
    green:  0x62f59a,    // BIOS
    blue:   0x33c9ff,    // GLACIA
    purple: 0xb469ff,    // TOXIN
}

// gamma — 지배 색을 더 강하게 끌어당기기 위한 보정 (1.0 = 단순 평균)
function blendByBias(bias, gamma = 1.5) {
    let r = 0, g = 0, b = 0, total = 0;
    for (const [key, w] of Object.entries(bias)) {
        const ww = Math.pow(w, gamma);
        const c = ELEMENT_HEX[key];
        r += ((c >> 16) & 0xff) * ww;
        g += ((c >> 8)  & 0xff) * ww;
        b +=  (c        & 0xff) * ww;
        total += ww;
    }
    return rgb(r / total, g / total, b / total);
}
```

`gamma`로 지배 색 우세 정도를 조절. 권장 값:

| gamma | 효과                                                  |
| ----- | ----------------------------------------------------- |
| 1.0   | 단순 가중 평균 — 모든 색이 골고루 섞여 *탁한* 톤      |
| 1.5   | 권장 — 지배 색이 분명하면서 부재 원소도 살짝 비침    |
| 2.0~  | 거의 단색 — 행성 구분은 쉽지만 미묘한 차이가 사라짐   |

#### 화염 행성 합성 예시 (gamma = 1.5)

| 항목                            | 결과                       |
| ------------------------------- | -------------------------- |
| Bias                            | IGNIS 3.0 / ARC 0.8 / BIOS 0.6 / GLACIA 0.5 / TOXIN 0.7 |
| 단순 평균 (감마 1.0)            | `#d37a85` (탁한 더스티)    |
| 가중 합성 (감마 1.5)            | `#e56476` (붉은 분홍)      |
| **톤다운 후 환경 베이스 (권장)**| **`#c5848d` (먼지 낀 로즈)**|
| 액센트 (단일 IGNIS, 톤다운 X)   | `#ff3f5f`                  |

#### 채도 톤다운 — *구안(빈티지) 행성 톤*

행성 환경은 *세월이 지나 빛바랜* 듯한 가라앉은 톤을 기본으로 한다.
블렌드 결과를 그대로 두면 너무 또렷한 게임용 컬러가 되므로, HSL의
채도(S)와 명도(L)를 살짝 깎아 *구안 톤*을 만든다.

```
base = blendByBias(bias);                          // 가중 합성 색
base = adjustHsl(base, { satMul: 0.5, lightMul: 0.92 });
//   채도 절반, 명도 -8% → 구안 톤 환경 베이스
```

| satMul   | lightMul | 결과 무드                              |
| -------- | -------- | -------------------------------------- |
| 0.7      | 0.95     | 약하게 빛바랜 — 거의 게임 컬러 유지    |
| **0.5**  | **0.92** | **권장** — 빈티지 SF, 가라앉은 톤       |
| 0.3      | 0.88     | 묵직한 흑백에 가까운 톤 (탐사형 행성)   |

> **중요** — 채도 톤다운은 *환경 베이스*에만 적용. *액센트 / 글로우 /
> 결정 입자 / HUD 칩*은 원래 채도 그대로 두어, 가라앉은 배경 위에서
> 또렷하게 튀어 보이게 한다. UI 가독성도 이걸로 확보된다.

#### 팔레트 슬롯 분배

톤다운된 *base 색* 한 개를 환경 곳곳에 강도만 달리해 적용한다.

| 슬롯               | 추천 처리                                         | 톤다운? |
| ------------------ | ------------------------------------------------- | ------- |
| 하늘 / 배경 그라데 | `base` × 0.75 (어둡게) → `base` × 1.05 그라데     | O       |
| 지형 베이스        | `base` × 0.45 (더 가라앉음)                       | O       |
| 안개 / 분위기      | `base` × 0.85, 알파 0.4                           | O       |
| 태양 / 별          | 액센트 × 0.9                                      | O (약)  |
| 액센트 / 글로우    | *지배 단일 색* (블렌드 없이, 채도 그대로)         | **X**   |
| 적 결정 입자       | 결정마다 자기 *원소 단일 색* (드랍 색과 일치)     | **X**   |
| HUD 칩 / 진척 바   | 네온 테마 그대로 (`neon-theme.js`)                | **X**   |

→ "환경은 빛바랜 구안 톤, 자원·UI·글로우는 또렷한 네온" — 시각 위계가
   자연스럽게 분리되고, *드랍된 결정이 풍경 위에서 빛나는* 픽업 매력이
   살아난다.

#### 적용 흐름

1. 행성 정의에서 `bias` 만 지정 (단일 진실 공급원).
2. 로딩 시 `blendByBias(bias, 1.5)` → 가중 합성 색.
3. `adjustHsl(..., { satMul: 0.5, lightMul: 0.92 })` → 톤다운 베이스.
4. 베이스를 *환경 슬롯*에만 분배. 액센트/입자/UI는 원소 단일 색 사용.
5. 손으로 잡고 싶을 때만 `paletteOverride: { ... }` 으로 덮어씀.

## 4. 재화 활용처

자원은 *단일 사용*과 *조합 사용* 두 트랙을 가진다.

### 4-A. 거점 출발 코어 (메인 미션)

기존 `fuelCapacity = 3` 슬롯을 5종 결정 코어로 확장.

```
┌── DEPARTURE CORE ──┐
│ [IGNIS]  ▓▓▓░░ 3/5 │   화염 분사 안정도
│ [ARC]    ▓▓░░░ 2/5 │   항법 및 시동
│ [BIOS]   ▓▓▓▓░ 4/5 │   생명 유지 시스템
│ [GLACIA] ▓░░░░ 1/5 │   엔진 냉각
│ [TOXIN]  ░░░░░ 0/5 │   추진 가스 압력
└────────────────────┘
```

- **출발 가능 조건**: 모든 코어 ≥ 1.
- **출발 안정도**: 코어 합계가 높을수록 카운트다운 중 스폰량 감소.
- **불균형 패널티**: 한 코어가 0이면 강제 실패 / 1이면 카운트 ×1.5.
- 한 종으로만 채울 수 없게 만들어 *5종 모두 모으는 동기*를 부여.

### 4-B. 무기 강화 (검 / 추후 스킬)

| 강화 트랙       | 주 자원 | 부 자원 | 효과 예시                           |
| --------------- | ------- | ------- | ----------------------------------- |
| 화염 부여       | IGNIS×4 | ARC×1   | 검 공격 시 폭발 잔여, +30% 데미지   |
| 사거리 확장     | ARC×3   | GLACIA×1| `sword.range` 단계적 +0.3           |
| 흡혈            | BIOS×4  | TOXIN×1 | 처치 시 HP 회복                     |
| 빙결            | GLACIA×3| BIOS×1  | 적중 시 적 둔화 1초                 |
| 부식 검         | TOXIN×4 | IGNIS×1 | DoT, 방어력 무시 일부               |

→ `config.js sword.*` 의 수치들을 단계적으로 끌어올리는 *영구 업그레이드*.

### 4-C. 거점 시설 / 패시브

| 시설        | 자원 비율               | 효과                                  |
| ----------- | ----------------------- | ------------------------------------- |
| 자동 포탑   | IGNIS+ARC               | 거점 근처 자동 사격                   |
| 회복 노드   | BIOS+GLACIA             | 범위 내 HP 리젠                       |
| 가스 트랩   | TOXIN+IGNIS             | 진입 적 광역 DoT                      |
| 방벽        | GLACIA+BIOS             | 거점 외곽 쉴드 게이지                 |
| 신호 송출기 | ARC+TOXIN               | 다음 출발 시 보상 ×1.2                |

거점 강화는 *판 사이*에 누적되는 메타 진척으로 설계.

### 4-D. 합성 / 변환

부족한 자원을 메우기 위한 *2:1 변환*. 균형을 너무 무너뜨리지 않도록
손해 비율을 강제.

```
IGNIS 2 + ARC 2  → IGNIS 1 (정제 손실 50%)
BIOS  3          → GLACIA 1 (특수 장치 필요)
```

### 4-E. 일회성 소비

전투 중 즉시 소모하는 *전술 아이템*도 같은 자원 풀을 공유:

- IGNIS×3 → 화염 폭탄 던지기
- GLACIA×3 → 자기 보호막 5초
- BIOS×3 → 즉시 회복 50%
- TOXIN×3 → 광역 약화 5초
- ARC×3 → 다음 검 휘두름 체이닝 ×3

## 5. 자원 가치 기준

밸런싱 시 *각 자원이 시간당 얼마 들어오는지*를 기준점으로 잡는다.

| 자원   | 일반 드랍 빈도 (예상) | 1유닛 가치 기준 |
| ------ | --------------------- | --------------- |
| IGNIS  | 보통                  | 1.0×            |
| ARC    | 보통                  | 1.0×            |
| BIOS   | 약간 낮음             | 1.2×            |
| GLACIA | 보통                  | 1.0×            |

업그레이드 비용은 위 가중치를 곱해 산출. 희귀 자원으로 *업그레이드 트리
중후반*을 게이팅.

## 8. 코드 작업 메모

- `home-controller.js` `ores: { red, yellow, green, blue, purple }`은
  그대로 사용. 키 이름은 색 기준 유지하되, 표시 라벨과 효과명만 원소
  테마(IGNIS/ARC/BIOS/GLACIA/TOXIN)로 노출한다.
- `config.js drops.*` 에 자원별 드랍 확률/색/이펙트 컬러 추가 예정.
- 적 타입 → 드랍 풀 매핑은 `enemy` 정의 측에 `dropPool` 필드 추가.
- HUD 광석 칩(`hud.js`)은 이미 5색 + 카운터 구조 → 그대로 재사용 가능.
- **행성 정의 (`config.js planets[*]`)** — 신규 추가 권장:
  ```
  planets: {
      ember:   { color: 0xff3f5f, dominant: 'red',    bias: { red:3.0, yellow:0.8, green:0.6, blue:0.5, purple:0.7 } },
      storm:   { color: 0xffd84f, dominant: 'yellow', bias: { ... } },
      jungle:  { color: 0x62f59a, dominant: 'green',  bias: { ... } },
      glacier: { color: 0x33c9ff, dominant: 'blue',   bias: { ... } },
      mire:    { color: 0xb469ff, dominant: 'purple', bias: { ... } },
  }
  ```
- 행성 전환은 현재 `home-controller.js`의 `success` 분기를 *세션 종료
  + 행성 선택 화면 → 새 세션 시작* 으로 확장.
- 환경 톤(하늘색, 지형 베이스 컬러, 안개 색)은 `world.bgColor` 등을
  행성별로 스왑하는 형태로 구현.
- **팔레트 합성 헬퍼** — `src/js/world/planet-palette.js` (신규):
  ```
  export const ELEMENT_HEX = {
      red: 0xff3f5f, yellow: 0xffd84f, green: 0x62f59a,
      blue: 0x33c9ff, purple: 0xb469ff,
  };
  export function blendByBias(bias, gamma = 1.5) { /* 위 공식 */ }
  export function adjustHsl(hex, { satMul = 1, lightMul = 1 }) { /* HSL 변환 후 보정 */ }

  // 환경용 구안 톤 베이스 + 순수 액센트의 분리된 팔레트
  export function planetPalette(planetCfg) {
      const blended = blendByBias(planetCfg.bias);
      const base    = adjustHsl(blended, { satMul: 0.5, lightMul: 0.92 }); // 구안 톤
      const accent  = ELEMENT_HEX[planetCfg.dominant]; // 액센트는 톤다운 X

      return {
          sky:      mul(base, 0.95),
          terrain:  mul(base, 0.45),
          fog:      mul(base, 0.85),
          accent,                         // 글로우/입자/UI 강조용
          sun:      mul(accent, 0.9),
      };
  }
  ```
- 행성 진입 시 `world.applyPalette(planetPalette(cfg))` 한 번 호출로
  씬의 모든 색 슬롯을 갈아끼우는 구조 권장.
